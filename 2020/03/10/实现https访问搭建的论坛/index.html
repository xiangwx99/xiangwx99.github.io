实现https访问搭建的论坛

Linux
发布日期:   2020-03-10
文章字数:   725
阅读时长:   3 分
阅读次数:  

对于刚搭建好的论坛，无法基于https安全访问，所以以下提供一种解决方法

第一步：首先搭建discuz论坛

可以参考：linux就该这么学搭建好论坛此时的论坛只能够基于http访问，而不可以基于https访问

第二步：使用OpenSSL生成免费的证书

2.1生成一个RSA私钥

$ openssl genrsa -des3 -out wahaha.key 1024

2.2创建证书签名请求CSR文件

openssl req -new -key wahaha.key -out wahaha.csr
Enter pass phrase for wahaha.key:  //输入密码
You are about to be asked to enter information that will be incorporated
into your certificate request.
What you are about to enter is what is called a Distinguished Name or a DN.
There are quite a few fields but you can leave some blank
For some fields there will be a default value,
If you enter '.', the field will be left blank.
Country Name (2 letter code) [XX]:CN  //输入国家代码,中国为CN
State or Province Name (full name) []:SiChuan  //输入所在省份
Locality Name (eg, city) [Default City]:ChengDu  //输入城市
Organization Name (eg, company) [Default Company Ltd]:stu  //输入机构组织，我随便写的stu
Organizational Unit Name (eg, section) []:stu  //输入机构部门
Common Name (eg, your name or your server's hostname) []:*.abc.com  //输入域名
Email Address []:admin@abc.com  //输入邮箱地址           

Please enter the following 'extra' attributes
to be sent with your certificate request
A challenge password []: //输入证书密码，如不想输入直接回车即可
An optional company name []:  //直接回车

2.3查看文件是否生成成功

2.4拷贝一个不需要输入密码的密钥文件

# openssl rsa -in wahaha.key -out wahaha_nopass.key

2.5生成CA证书

#openssl x509 -req -days 365 -in wahaha.csr -signkey wahaha.key -out wahaha.crt


X509:指定格式
-in：指定请求文件
-signkey:自签名

查看证书是否生成成功:

第三步：nginx配置SSL证书实现https服务

3.1将上述过程中中生成的证书，移动到nginx配置文件的同级目录中，创建一个名为cert的文件夹存放证书，并且查看是否移动成功

#mkdir /usr/local/nginx/conf/cert
#mv wahaha
#mv wahaha.crt /usr/local/nginx/conf/cert
#mv wahaha.key /usr/local/nginx/conf/cert
#mv wahaha.csr /usr/local/nginx/conf/cert
#mv wahaha_nopass.key /usr/local/nginx/conf/cert
#cd /usr/local/nginx/conf/cert

3.2使用vim编辑nginx配置文件nginx.conf，增加https协议

98行附近有个 # HTTPS server将下面的注释全部去掉

3.4重启Nginx

#/etc/rc.d/init.d/nginx restart


若重启失败就要考虑nginx.conf中修改是否出错

3.5使用https访问本地搭建好的论坛



3.6访问时下载PHP文件的格式，可以得知https访问不支持PHP格式于是修改配置文件nginx.conf添加下面代码（添加到刚才解除注释的server{}里面）

 location ~ \.php$ {
            root           html;
            fastcgi_pass   127.0.0.1:9000;
            fastcgi_index  index.php;
            fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
            include        fastcgi_params;
        }

3.7重启Nginx

#/etc/rc.d/init.d/nginx restart

若重启失败就要考虑nginx.conf中修改是否出错

3.8使用https访问本地搭建好的论坛

访问成功
